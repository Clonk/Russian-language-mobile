Инструкция по компиляции собственного мобильного приложения средствами forge-tools.

1)Скачать и установить фреймворк с сайта производителя https://trigger.io/ , при отсутствии python, скачать и установить в том числе и его - http://www.python.org/getit/ (строго версию 2.7).
2)Пройти в каталог установки и запустить файл go.bat
3)В командной строке выберите необходимую директорию (командой cd перемещайтесь по директориям, командой mkdir создавайте новые каталоги), и находясь в ней, введите forge create.
4)После принятия команды вас попросят ввести имя вашей программы. При создании могут возникнуть не фатальные, но всё же ошибки, связанные с кодировкой. Плюйте на них.
5)В выбранном каталоге появится папка src - это и есть исходные страницы, которые в будущем будут компилироваться в мобильное приложение.
6)Ограничиваясь функциональностью html5, javascript, css и php вы создаёте ваше приложение, состоящее из html или php страниц (по сути представляет из себя простенький сайт), после чего сохраняете всё в папке src.
7)Теперь при помощи go.bat заходите в каталог, в котором создали ваше приложение (именно в нём находится папка src), выполните команду forge build.
8)После выполнения в каталоге программы появится папка development, в ней находятся скомпонованные и готовые к компиляции файлы, разделённые по назначению: chrome, android, ios, web.
9)В процессе компоновки файлов из папки src в папку development может сломаться отображение кириллического текста, во избежание подобных проблем, можно поиграться с разной кодировкой страниц, или же вручную заменить файлы из папки development соответствующими файлами из папки src (проверено – костыль работает!).
11)Теперь, введя команду forge run [название назначения, например android] вы сможете отладить ваше приложение на эмуляторе (в случае android, forge предложит скачать необходимый sdk для эмуляции). Надо заметить, что при тестировании программы на эмуляторе android, следует использовать версию прошивки эмулируемого устройства 2.2, т.к. 2.3 имеет проблемы с рендерингом при тестировании.
12)После отладки вашего приложения пришло время его публикации! Для этого достаточно ввести в командную строку команду forge package [назначение, напр. android]. Forge запросит адрес размещения вашего keystore, его псевдоним и пароль. Если у вас нет данного файла, создайте его при помощи keytool.exe, входящего в комплект поставки java (если же до сих пор у вас не стоит java, не ясно как вы добрались до этого пункта инструкции, идите и ставьте).
13)После присоединения файла keystore командная строка выдаст ещё одну порцию отчётов, а в папке приложения появится каталог release, в нем-то и находится свежеиспечённый файл, готовый к работе на целевом устройстве. Поздравляю!
